# nitrogen_cli ⚡

**CLI tool for Nitrogen plugins.** Scaffold, generate, and validate Nitrogen FFI plugins from the command line — no `build_runner` required in your project.

---

## Installation

```sh
dart pub global activate nitrogen_cli
```

Then add the Dart pub global bin to your `PATH` (one-time setup):

```sh
# Add to ~/.zshrc or ~/.bashrc:
export PATH="$PATH:$HOME/.pub-cache/bin"
```

---

## Commands

### `nitrogen generate`

Scans the current package for `*.native.dart` files and regenerates all outputs.

```sh
# From your plugin root:
nitrogen generate

# Or via pub global run:
dart pub global run nitrogen_cli:nitrogen generate
```

**What it produces (per `*.native.dart` file):**

| Output | Description |
|---|---|
| `*.g.dart` | Dart FFI implementation class |
| `*.bridge.g.kt` | Kotlin JNI bridge + `HybridXxxSpec` interface |
| `*.bridge.g.swift` | Swift `@_cdecl` bridge + `HybridXxxProtocol` protocol |
| `*.bridge.g.h` | C header with `extern "C"` declarations |

All files begin with `// Generated by Nitrogen Modules. Do not edit.` — it is safe to commit them to VCS.

### `nitrogen init <PluginName>` *(coming soon)*

Scaffolds a complete Nitrogen plugin from scratch:

```sh
nitrogen init MyCameraPlugin
```

Creates:
```
my_camera_plugin/
├── pubspec.yaml                          ← pre-wired with nitro + nitrogen
├── lib/src/my_camera_plugin.native.dart  ← starter spec
├── android/src/main/kotlin/.../Impl.kt  ← stub implementation
├── ios/Classes/Impl.swift               ← stub implementation
├── android/CMakeLists.txt               ← wired for JNI
└── README.md                            ← usage guide
```

---

### `nitrogen doctor` *(coming soon)*

Validates your plugin setup:

```sh
nitrogen doctor
```

For each `*.native.dart` in the project:
- ✅ Checks generated files exist and are up to date
- ✅ Runs spec validator (annotation rules, type compatibility)
- ✅ Verifies lib names match `pubspec.yaml` `ffiPlugin` declarations
- Reports `OK / STALE / ERROR` per spec file

---

### `nitrogen clean` *(coming soon)*

Deletes all generated files:

```sh
nitrogen clean
```

Removes: `*.g.dart`, `*.bridge.g.kt`, `*.bridge.g.swift`, `*.bridge.g.h`

## Quick start — 5 minutes to a working plugin

### 1. Activate the CLI
```sh
dart pub global activate nitrogen_cli
```

### 2. Create your spec file
```dart
// lib/src/greeter.native.dart
import 'package:nitro/nitro.dart';
part 'greeter.g.dart';

@NitroModule(ios: NativeImpl.swift, android: NativeImpl.kotlin)
abstract class Greeter extends HybridObject {
  static final Greeter instance = _GreeterImpl(NitroRuntime.loadLib('greeter'));

  String greet(String name);

  @nitroAsync
  Future<String> greetAsync(String name);
}
```

### 3. Generate
```sh
nitrogen generate
# ✅ Nitrogen generation complete!
```

### 4. Implement on Android

```kotlin
// android/.../GreeterImpl.kt
class GreeterImpl : HybridGreeterSpec {
  override fun greet(name: String): String = "Hello, $name!"
  override suspend fun greetAsync(name: String): String {
    delay(500)
    return "Hello async, $name!"
  }
}

class GreeterPlugin : FlutterPlugin {
  override fun onAttachedToEngine(binding: FlutterPluginBinding) {
    GreeterJniBridge.register(GreeterImpl())
  }
  override fun onDetachedFromEngine(binding: FlutterPluginBinding) {}
}
```

### 5. Implement on iOS

```swift
// ios/Classes/GreeterImpl.swift
public class GreeterImpl: NSObject, HybridGreeterProtocol {
  public func greet(name: String) -> String { "Hello, \(name)!" }
  public func greetAsync(name: String) async throws -> String {
    try await Task.sleep(nanoseconds: 500_000_000)
    return "Hello async, \(name)!"
  }
}

public class GreeterPlugin: NSObject, FlutterPlugin {
  public static func register(with registrar: FlutterPluginRegistrar) {
    GreeterRegistry.register(GreeterImpl())
  }
}
```

### 6. Use it in Flutter

```dart
final greeting = Greeter.instance.greet('World');
print(greeting); // Hello, World!

final asyncGreeting = await Greeter.instance.greetAsync('Flutter');
print(asyncGreeting); // Hello async, Flutter!
```
