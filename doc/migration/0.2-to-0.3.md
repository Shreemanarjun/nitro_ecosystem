# Migration Guide: Nitro 0.2 → 0.3

This guide covers every breaking change between the 0.2 and 0.3 releases.
Run `nitrogen generate` and `nitrogen link` after completing the steps below.

---

## Summary of breaking changes

| Area | Change | Action required |
|---|---|---|
| `@NitroModule` | New typed platform fields (`macos`, `windows`, `linux`, `web`) | No action — old `ios`/`android` shorthand still works |
| `NativeImpl` | Sealed hierarchy with typed platform markers | No action — `NativeImpl.swift` etc. are unchanged const values |
| `BridgeParam.defaultLiteral` | New field — generators now propagate default values | Run `nitrogen generate` to regenerate |
| Struct release | `${lib}_release_${StructName}` now called via `NativeFinalizer` | Remove any manual `release()` calls in Dart |
| `@NitroOwned` | New annotation on `NativeHandle<T>` returns | See §NativeHandle below |
| `@ZeroCopy` return | New: `@ZeroCopy` on `Uint8List` function returns | No action unless you used the workaround |
| Generator architecture | Generators moved to `languages/*` bundles | No user-facing change |
| `NitroRuntime.callAsync<T>` | Now typed (`<T>`) instead of `dynamic` | Regenerate; hand-written callers need `<T>` |
| ABI version check | `${lib}_nitro_abi_version()` called on init | Regenerate all bridges; stale `.so` → `StateError` |
| JNI error slot | Now thread-local (`thread_local NitroError`) | Regenerate; no Dart-side change |

## Step-by-step migration

### 1. Update `pubspec.yaml`

```yaml
dependencies:
  nitro: ^0.3.0

dev_dependencies:
  build_runner: ^2.4.0
  nitro_generator: ^0.3.0
  nitrogen_cli: ^0.3.0
```

Run `flutter pub get`.

### 2. Regenerate all bridges

```bash
cd your_plugin/
dart run nitrogen_cli generate
dart run nitrogen_cli link
```

This regenerates `.g.dart`, `.bridge.g.kt`, `.bridge.g.swift`, `.bridge.g.cpp`, and `.bridge.g.h`.

### 3. Optional: adopt typed platform fields

0.3 introduces typed platform markers for compile-time safety:

```dart
// Old (still works — backward compatible):
@NitroModule(ios: NativeImpl.swift, android: NativeImpl.kotlin)

// New (explicit, catches mismatches at compile time):
@NitroModule(
  ios:     AppleNativeImpl.swift,
  macos:   AppleNativeImpl.swift,
  android: AndroidNativeImpl.kotlin,
  windows: WindowsNativeImpl.cpp,
  linux:   LinuxNativeImpl.cpp,
)
```

The old shorthand `NativeImpl.swift`, `NativeImpl.kotlin`, `NativeImpl.cpp` etc. remain valid — they return the same const objects as the typed forms.

### 4. Remove manual struct release calls (if any)

In 0.2, some apps called a manual `releaseStruct(ptr)` helper. In 0.3 release is automatic via `NativeFinalizer`:

```dart
// 0.2 pattern — remove this:
final myStruct = module.getStruct();
// ... use myStruct ...
module.releaseStruct(myStruct); // ← delete

// 0.3 — automatic cleanup via NativeFinalizer when struct is GC'd.
final myStruct = module.getStruct();
// No manual release needed.
```

### 5. NativeHandle<T> (new in 0.3)

If you were passing raw `Pointer<Void>` across the bridge manually, migrate to `NativeHandle<T>`:

```dart
// 0.2 — manual pointer plumbing (error-prone):
// abstract class Camera extends HybridObject {
//   int getFrameAddress();
// }
// final addr = cam.getFrameAddress();
// final ptr = Pointer<Void>.fromAddress(addr);

// 0.3 — zero-codec NativeHandle:
abstract class Camera extends HybridObject {
  @NitroOwned
  NativeHandle<Void> acquireFrame();
  void processFrame(NativeHandle<Void> handle);
}
// Dart:
final frame = cam.acquireFrame();      // NativeHandle<Void>
final ptr = frame.pointer.cast<FrameNative>();
frame.release();                       // or let GC handle it
```

See [native_handle.md](../advanced/native_handle.md) for full details.

### 6. Nullable optional params — sentinel encoding

If you were manually checking `-1` or `NaN` on the native side, those sentinels are unchanged in 0.3. No migration needed. Default values (`{int x = 5}`) are now propagated correctly — regenerate to pick up the fix.

### 7. ABI version mismatch error

After updating, if you see:

```
StateError: nitro_camera ABI version mismatch — regenerate and re-link.
```

It means your `.so` was compiled against 0.2 headers. Run `nitrogen generate` then rebuild the native library (`flutter run` or `flutter build apk`).

## Feature additions in 0.3 (no migration needed)

- `@NitroNativeAsync` — zero-hop async path posting results via `Dart_PostCObject_DL`
- `NitroConfig.timelineTracingEnabled` — opt-in `Timeline` spans around bridge calls
- `@ZeroCopy` on TypedData function returns — avoids `GetByteArrayRegion` copy
- All callback parameter types now bridge correctly (`double`, `bool`, `String`, `@HybridStruct`, `@HybridRecord`)
- `IsolatePool` min-heap scheduling — ~30 % lower dispatch latency under load
- `ZeroCopyBuffer` and its 9 typed variants now have `NativeFinalizer`-backed automatic release
